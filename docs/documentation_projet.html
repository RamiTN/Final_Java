1. Aperçu de l'Architecture du Projet

Comment toutes les couches sont organisées et communiquent entre elles

Ce projet suit un modèle classique MVC (Modèle-Vue-Contrôleur) enrichi par une couche DAO (Data Access Object) et une couche Service. Cela permet à chaque morceau de code de se concentrer sur une seule tâche (Principe de Responsabilité Unique).

Responsabilités des Couches

CouchePackageRôle
ModèlemodelObjets Java simples (POJO) qui contiennent des données. Aucune logique complexe, juste des champs et des getters/setters.
Base de DonnéesdbCrée la connexion SQLite et initialise les tables au premier lancement.
DAOdaoToutes les requêtes SQL (CRUD) se trouvent ici. Chaque DAO gère une seule table et renvoie des objets du Modèle.
ServiceserviceLa logique métier : authentification, export PDF, appels à l'IA. Fait le pont entre les contrôleurs et les DAO.
ContrôleurcontrollerGère les événements de l'interface (clics de boutons). Appelle les Services/DAO et met à jour la Vue.
VueviewFichiers XML (FXML) qui définissent la mise en page visuelle. Liés aux Contrôleurs.
💡 Règle de conception clé : Les contrôleurs ne touchent JAMAIS directement à la base de données. Ils appellent un Service ou un DAO. Les DAO contiennent tout le SQL. Cela rend le code propre, testable et facile à maintenir.

2. Main.java – Point d'Entrée

La classe qui démarre toute l'application

classe

Main

Package: racine

Hérite de: javafx.application.Application

C'est la toute première classe exécutée au lancement. Elle effectue trois actions importantes dans l'ordre :

  1. Appelle DatabaseConnection.initializeDatabase() pour créer la base de données SQLite et les tables si elles n'existent pas encore.
  2. Charge la vue landing.fxml (page d'accueil) à l'aide de FXMLLoader.
  3. Configure la fenêtre principale (Stage), lui donne un titre, la met en plein écran et l'affiche.
MéthodeRetourDescription

main(String[] args)voidPoint d'entrée standard Java. Appelle launch(args) qui déclenche le start().

4. Couche Base de Données – DatabaseConnection.java

Le point de contact unique entre l'application et le fichier SQLite

db
classe

DatabaseConnection

Package : db  |  Fichier : cvcreator.db (SQLite)

Cette classe utilitaire a deux rôles : fournir une connexion à la base de données à ceux qui en ont besoin, et créer toutes les tables au premier lancement. Elle utilise le driver JDBC SQLite de Xerial.

MéthodeRetourDescription
getConnection()ConnectionStatique. Ouvre et renvoie une nouvelle connexion JDBC vers cvcreator.db. Chaque DAO appelle cette méthode à chaque fois qu'il doit exécuter une requête SQL. Utilise le try-with-resources pour fermer automatiquement les connexions.
initializeDatabase()voidStatique. Appelée une fois au démarrage par Main.java. Crée trois tables (users, cvs, jobs) en utilisant IF NOT EXISTS, ce qui permet de l'appeler à chaque lancement sans risque. Insère également le compte admin par défaut s'il n'existe pas.
🔑 Compte Admin par Défaut : Au premier lancement, si aucun utilisateur avec l'email admin@admin.com n'existe, le système en crée un automatiquement avec le mot de passe admin123 et le rôle admin. Cela garantit un accès au tableau de bord administrateur.

URL de Connexion

La chaîne de connexion est : jdbc:sqlite:cvcreator.db — cela signifie que SQLite cherchera (ou créera) un fichier nommé cvcreator.db dans le répertoire de travail (la racine du projet).

5. Couche DAO (Data Access Object)

Toutes les requêtes SQL y sont contenues. Chaque classe DAO gère une table.

Le modèle DAO sépare la logique de la base de données de la logique métier. Chaque classe DAO sait comment : Insérer, Mettre à jour, Supprimer et Trouver des enregistrements pour sa table. Elles utilisent toutes PreparedStatement pour prévenir les injections SQL.

6. Couche Service (Services)

La logique métier qui fait le pont entre les Contrôleurs et les DAO

Les services contiennent une logique trop complexe pour un DAO mais trop réutilisable pour être placée dans un contrôleur spécifique. Ils constituent le "cerveau" de l'application.

service

AuthService.java

But : Gère l'utilisateur actuellement connecté sous forme de session globale. Gère la logique de connexion et d'inscription.

Conception clé : currentUser est un champ static — il existe une seule fois pour toute la durée de vie de l'application. Cela agit comme une session simple en mémoire (typique pour une application de bureau).

MéthodeDescription
getCurrentUser()Renvoie l'utilisateur (User) actuellement connecté, ou null s'il n'y a pas de session.
logout()Efface la session en définissant currentUser = null.
login(String email, String password)Récupère l'utilisateur par email depuis la BD, puis compare le mot de passe. Si ça correspond, définit currentUser et renvoie l'utilisateur.
register(String name, String email, String password)Vérifie si l'email existe déjà. Si non, crée un nouvel utilisateur avec le rôle "user" et l'insère. Renvoie true en cas de succès.
⚠️ Note de Sécurité : Les mots de passe sont stockés en texte clair. Dans un système en production, il faudrait utiliser un algorithme de hachage comme BCrypt. C'est acceptable pour un projet universitaire.
service

CvService.java

But : Enveloppe les opérations du CVDAO et gère la fonctionnalité d'export PDF en utilisant Apache PDFBox.

MéthodeDescription
createCv(), updateCv(), deleteCv()Délèguent simplement les appels aux méthodes correspondantes du CVDAO.
getCvsByUser(int userId)Renvoie tous les CVs pour un ID utilisateur donné via CVDAO.findByUserId().
exportToPdf(Cv cv, String filePath)Méthode complexe. Utilise Apache PDFBox pour générer un PDF professionnellement formaté au chemin spécifié. Renvoie true en cas de succès.

Fonctionnement de exportToPdf()

  1. Crée un nouveau PDDocument avec une page A4.
  2. Dessine une barre d'en-tête bleu foncé en haut de la page.
  3. Écrit le nom complet (centré, grand texte blanc) et les infos de contact.
  4. Appelle addSection() pour chaque section : Objectif, Éducation, Expérience, Compétences, Langues.
  5. Chaque section a un titre coloré en gras, un soulignement fin, et un texte avec retour à la ligne automatique (word-wrap).
  6. Sauvegarde le document.
service

AiService.java

But : Envoie des requêtes textuelles à l'API Google Gemini 2.5 Flash et renvoie la réponse de l'IA. Utilisé pour les conseils CV, le coaching d'entretien et le chatbot.

MéthodeDescription
loadApiKey()Privée. Essaie de charger la clé API depuis : (1) la variable d'environnement système, puis (2) le fichier .env.
askGemini(String prompt)Méthode publique principale. Envoie le prompt à Gemini et renvoie la réponse textuelle. Gère les erreurs HTTP (400, 403, 429, 503) avec des messages clairs. Inclut une logique de "retry" (nouvel essai) en cas de surcharge de l'IA.
buildRequestBody(String prompt)Privée. Construit le corps JSON de la requête Gemini en utilisant Gson.
extractText(String json)Privée. Analyse la réponse JSON de Gemini et extrait le texte utile.
💡 Multithreading : askGemini() est appelé depuis un thread en arrière-plan (background Thread) dans les contrôleurs pour éviter de figer l'interface JavaFX. La réponse est ensuite renvoyée à l'interface via Platform.runLater().

Configuration de la clé API

Il faut créer un fichier nommé .env à la racine du projet avec cette ligne :

API_KEY=votre_cle_api_gemini_ici

7. Couche Contrôleur (Controllers)

Connexion des interfaces FXML aux Services et aux DAO

Chaque écran (fichier FXML) a une classe Contrôleur. Les contrôleurs répondent aux clics de boutons, lisent ce que l'utilisateur a tapé dans les formulaires, et mettent à jour les étiquettes et tableaux à l'écran.

controller

LandingController.java

Écran : landing.fxml (Le tout premier écran)

Rôle : Affiche un bel écran d'accueil. Il utilise les 3 DAO pour compter le total des utilisateurs, des CVs et des offres d'emploi afin d'afficher des statistiques dynamiques. Il charge aussi les 5 dernières offres d'emploi pour les afficher sous forme de cartes.

controller

UserController.java

Écrans : login.fxml, register.fxml, dashboard.fxml, profile.fxml

Rôle :

  • Auth : Gère la logique quand l'utilisateur clique sur Connexion ou Inscription. Valide les emails, mots de passe, et le format du nom. Appelle AuthService.
  • Tableau de bord : Remplit le tableau de bord avec les CVs récents, calcule un "% de Complétion du Profil", et affiche le nombre de jobs disponibles.
  • Recherche : Inclut un écouteur de recherche en temps réel qui filtre les CVs récents par nom, compétences ou objectif pendant la frappe.
  • Navigation : Contient de nombreuses méthodes goToX() pour charger les autres écrans.
controller

CVController.java

Écran : cv.fxml (L'Éditeur de CV)

C'est le plus gros contrôleur. Il gère un formulaire dynamique où les utilisateurs peuvent créer, modifier, supprimer et télécharger des CVs.

Fonctionnalités Clés :

  • Listes Dynamiques : L'utilisateur peut cliquer sur "Ajouter" pour générer de nouveaux champs texte (TextField) pour l'Éducation et l'Expérience. Le contrôleur crée dynamiquement ces éléments HBox en Java.
  • Tags de Compétences : L'ajout d'une compétence génère un petit "tag" bleu avec un bouton X pour le supprimer.
  • Validation de Formulaire : Vérifie la longueur du nom, le format de l'email (regex) et du numéro de téléphone avant la sauvegarde.
  • Conversion de Données : Prend les listes visuelles dynamiques et les rassemble en de longues chaînes de caractères (séparées par \n---\n) pour les donner au CvService.
  • Intégration IA : Possède un bouton qui envoie les données du formulaire actuel au AiService et demande à Gemini de suggérer des améliorations pour le CV.
  • Export PDF : Ouvre une boîte de dialogue (FileChooser) pour laisser l'utilisateur choisir où sauvegarder le PDF, puis appelle CvService.exportToPdf().
controller

JobGuideController.java

Écran : JobGuide.fxml (L'Assistant Carrière IA)

Une interface de chat propulsée par Gemini AI. Elle possède trois modes distincts :

  1. Chat Standard : Agit comme un coach carrière généraliste.
  2. Pratique d'Entretien : Une boucle interactive. L'IA pose une question prédéfinie ("Parlez-moi de vous"). L'utilisateur tape une réponse. L'IA évalue la réponse, donne un retour et un exemple, et demande à l'utilisateur de réessayer. L'IA passe à la question suivante seulement si l'utilisateur tape "next".
  3. Conseil CV : Lit le premier CV de l'utilisateur en base de données et demande à l'IA de l'analyser.
  4. Job Matcher (Sans IA) : Un algorithme local qui prend les compétences du CV et vérifie si elles existent dans la description/exigences des offres d'emploi.

Utilise le multithreading pour que l'interface ne gèle pas en attendant l'API Gemini.

controller

AdminController.java

Écran : AdminDashboard.fxml

Accessible uniquement aux utilisateurs avec le rôle "admin". Gère tout le système :

  • Gestion des Utilisateurs : TableView de tous les utilisateurs. L'admin peut supprimer des comptes (sauf le sien).
  • Gestion des Emplois : Formulaires CRUD (Créer, Lire, Mettre à jour, Supprimer) complets. Inclut une validation d'URL pour les liens externes.
  • Graphiques : Utilise JavaFX PieChart pour montrer le ratio Admins/Utilisateurs normaux, et BarChart pour afficher le nombre d'offres par entreprise.
controller

JobController.java

Écran : Job.fxml

Un contrôleur très simple (lecture seule) pour les utilisateurs normaux. Il charge toutes les offres d'emploi depuis JobOfferDAO dans une TableView pour pouvoir les parcourir.

controller

InfoController.java

Écrans : features.fxml, how_it_works.fxml

Un contrôleur purement navigationnel utilisé par les pages d'information statiques. Ne contient que des méthodes goToX().

9. Flux de l'Application et Parcours Utilisateur

  1. Lancement : Main.java s'assure que la base de données existe. Affiche la Page d'Accueil (Landing Page).
  2. Page d'Accueil : Affiche les statistiques et les emplois récents. L'utilisateur clique sur "Se connecter".
  3. Connexion : L'utilisateur se connecte. AuthService sauvegarde la session.
    • Si Rôle == Admin ➔ Va vers le Tableau de Bord Admin (Gérer utilisateurs & jobs).
    • Si Rôle == User ➔ Va vers le Tableau de Bord Utilisateur.
  4. Tableau de Bord : Affiche les CVs récents et le taux de complétion du profil. L'utilisateur clique sur "Créer un CV".
  5. Éditeur de CV : L'utilisateur remplit les formulaires, ajoute des lignes dynamiques pour l'éducation/expérience, ajoute des tags pour les compétences.
    • Action : Clique sur AI Advice ➔ Gemini analyse les champs et donne des conseils.
    • Action : Clique sur Save ➔ Sauvegarde dans la BD SQLite.
    • Action : Clique sur Export PDF ➔ PDFBox génère un fichier PDF formaté.
  6. Job Guide (Assistant Emploi) : L'utilisateur clique sur l'icône IA.
    • Discute avec Gemini pour des conseils de carrière.
    • S'entraîne avec une boucle d'entretien interactive.
    • Obtient des correspondances avec les offres d'emploi internes basées sur les compétences de son CV.
